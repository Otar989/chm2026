## Что это и как устроено

- Весь сайт — это **один файл `index.html`**. Внутри него и вёрстка (HTML), и стиль (CSS), и логика (JavaScript). Никаких сборок, фреймворков, node_modules и серверной части.
- Сайт **статический**: GitHub Pages просто отдаёт `index.html`. Но он «живой» — все данные подгружаются **прямо в браузере** пользователя из публичных API и обновляются сами.
- Из-за этого **сайт не устаревает**: даже если файл не трогать неделями, счёт матчей, таблицы и новости будут актуальными — потому что они тянутся в реальном времени при каждом открытии страницы.
- Менять код нужно только когда хочешь **добавить/изменить функцию или дизайн**, а не «обновить данные».

```
chm2026/
├── index.html   ← весь сайт (вёрстка + стили + логика + резервный снимок данных)
└── README.md    ← этот файл
```

## Откуда берутся данные (живые)

Все источники — бесплатные публичные API, без ключей:

| Что | Источник | Как часто обновляется |
|---|---|---|
| Счёт, статус, расписание, таблицы | ESPN `site.api.espn.com/.../fifa.world/scoreboard` | каждые 40 сек |
| События матча (голы, карточки, замены), статистика, «сейчас на поле» | ESPN `.../summary?event={id}` | каждые 18 сек для идущих матчей |
| Новости (список) | ESPN `.../fifa.world/news?limit=50` | каждые 5 мин |
| Полный текст новости | ESPN `content.core.api.espn.com/v1/sports/news/{id}` | по клику на новость |
| Перевод новостей на русский | MyMemory `api.mymemory.translated.net` | по мере загрузки, с кэшем |

- Если прямой запрос к ESPN не проходит (блокировка CORS у конкретного провайдера/браузера) — включается **резервный прокси** (`allorigins`, `corsproxy`), чтобы данные всё равно загрузились.
- Если данные совсем недоступны — сайт показывает **сохранённый снимок** (зашит в `index.html`) и помечает это в шапке («снимок»), а не падает.

Индикатор в шапке:
- 🔴 **LIVE** — прямо сейчас идёт матч;
- 🟢 **онлайн · ЧЧ:ММ** — данные свежие, матчей сейчас нет;
- ⚪ **снимок** — нет связи с ESPN, показан сохранённый набор.

Время на всём сайте — **по Москве (МСК)**.

## Разделы сайта

- **Обзор** — большой герой-блок: фавориты на титул, ближайший/идущий матч, ключевые цифры (сыграно, голы, голов за матч, дней до финала), прогресс турнира, форма фаворитов, лучшая атака/оборона, лента последних результатов.
- **Лайв** — геймифицированное табло: идущие матчи с таймером по секундам, полосой прогресса матча, лентой событий, и анимацией **«⚽ ГОЛ!»** с конфетти при изменении счёта.
- **Матчи** — календарь по дням. По умолчанию открывается на **сегодня и далее**, прошлые матчи — по кнопке «Все матчи». У каждого матча — прогноз счёта.
- **Звёзды** — профиль Криштиану Роналду + карточки топ-игроков в стиле FUT. Клик по карточке открывает подробный профиль (как играет, статы, факты).
- **Новости** — живая лента ESPN, авто-разложенная по категориям (Матчи / Звёзды / Сборные / Слухи / Сенсации), переведённая на русский. Клик по новости — **полный текст статьи** с переводом.
- **Прогнозы** — шансы на титул, сетка плей-офф, гонка бомбардиров, награды, кто выйдет из групп, прогноз счёта на каждый матч, шансы дойти до стадии, смелые прогнозы.
- **Ставки** — личная корзина «что я бы поставил» с распределением банка, аутрайты, прогнозы на матчи. (Аналитика, не финансовый совет; 18+.)
- **Таблицы** — положение в 12 группах, считается вживую из реальных результатов.

Прогнозы, ставки и профили игроков — это **аналитика/редактура** (моё мнение), а факты (счёт, расписание, таблицы, новости) — **живые**.

## Как обновляется и деплоится (авто-деплой)

**Деплой полностью автоматический.** GitHub Pages пересобирает сайт **при каждом коммите в ветку `main`**.

Цепочка:

```
правка index.html  →  коммит в main  →  GitHub Pages сам собирает  →  сайт обновлён (~1 мин)
```

- Ничего нажимать для деплоя не нужно: как только новый `index.html` попал в `main`, через ~минуту он уже живой на https://otar989.github.io/chm2026/.
- Проверить статус деплоя: в репозитории справа блок **Deployments → github-pages**, либо вкладка **Actions**.
- Настройка Pages: **Settings → Pages → Source: Deploy from a branch → main / (root)**. Менять не нужно — уже настроено.

## Как править через облако

Поскольку проект в GitHub, его можно редактировать с **любого устройства, включая телефон**, без локального компьютера. Любой вариант ниже после коммита в `main` запускает авто-деплой.

### Вариант A — Codex / ИИ-агент (с телефона тоже)
1. Открой Codex (web или приложение ChatGPT) → подключи свой GitHub.
2. Выбери репозиторий **Otar989/chm2026**.
3. Дай задачу словами (например: «добавь раздел с расписанием трансляций»).
4. Codex внесёт правки и сделает **коммит** или **Pull Request**.
5. Если PR — открой его на GitHub и нажми **Merge** (влить в `main`).
6. Через ~минуту сайт обновится сам.

### Вариант B — приложение GitHub Mobile (телефон)
1. Установи **GitHub** из App Store / Google Play, войди.
2. Открой репозиторий → файл `index.html` → ✏️ (карандаш) → правка → **Commit**.

### Вариант C — веб-редактор github.dev (десктоп-браузер)
1. Зайди на репозиторий и нажми клавишу **`.`** (точка) — откроется VS Code в браузере.
2. Правь `index.html` → слева вкладка Source Control → Commit & Push в `main`.

### Вариант D — прямо на github.com (любой браузер)
1. Открой `index.html` в репозитории → ✏️ **Edit** → правка → **Commit changes**.
2. Или **Add file → Upload files**, чтобы заменить файл целиком новой версией.

> Совет: для крупных правок удобнее всего Codex/агент или github.dev. Для мелких — кнопка ✏️ прямо на сайте GitHub.

## Как править локально

1. Скачай/клонируй репозиторий:
   ```bash
   git clone https://github.com/Otar989/chm2026.git
   ```
2. Открой `index.html` в редакторе и правь. Чтобы посмотреть — просто открой файл в браузере (двойной клик) — всё работает, ведь это один самодостаточный файл.
3. Залей изменения:
   ```bash
   git add index.html
   git commit -m "что изменил"
   git push
   ```
4. GitHub Pages задеплоит автоматически.

## Технические детали

- **Один файл, без зависимостей.** Раньше использовался 3D-кубок на Three.js (CDN) — его убрали в пользу надёжного **векторного (SVG) кубка**, который рисуется всегда и не зависит от интернета/библиотек.
- **Время — МСК.** Всё отображение времени переведено в `Europe/Moscow` независимо от часового пояса устройства.
- **Живой таймер матча.** ESPN отдаёт время без секунд — секунды «докручиваются» локально и пересинхронизируются при каждом обновлении данных.
- **Перевод новостей.** Через MyMemory (бесплатно, без ключа), с кэшем в `localStorage`, чтобы не переводить одно и то же повторно. Если перевод недоступен — показывается оригинал (английский).
- **Имена игроков в событиях** вытаскиваются из текста ESPN (там они внутри английского описания), а не из отдельного поля — поэтому видно, кто забил и кого заменили.
- **Устойчивость к сбоям сети** — `fetchJSON` пробует прямой запрос, затем прокси; при полном отказе — зашитый снимок данных.

## Частые вопросы и решение проблем

**Сайт показывает «снимок», а не live.**
У конкретного браузера/провайдера заблокирован прямой доступ к ESPN, и прокси тоже не сработал. Обычно помогает обновление страницы. Это не ломает сайт — просто показывает сохранённые данные.

**Поменял файл, а на сайте старое.**
Браузер закэшировал. Жёсткое обновление: **Cmd/Ctrl + Shift + R**. И проверь, что коммит реально попал в `main` и деплой в **Actions** прошёл (зелёная галка).

**Новости/перевод не грузятся.**
MyMemory имеет дневной лимит у анонимов. Заголовки/тексты, что уже переведены, лежат в кэше и работают; новые могут на время остаться на английском.

**Хочу красивый адрес вместо otar989.github.io/chm2026.**
Можно подключить свой домен: **Settings → Pages → Custom domain**. Нужен купленный домен и DNS-настройка.
